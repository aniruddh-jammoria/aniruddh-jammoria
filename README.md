Currently building:
- **[renpho-health-mcp](https://github.com/aniruddh-jammoria/renpho-health-mcp)** - Pulls Renpho smart-scale readings (weight, body fat, muscle, water, BMI) out of the Renpho app into a local, append-only log, and ships an MCP server so an AI agent can answer questions about your history conversationally.
- **[obsidian-agentic-todolist (AgentBoard)](https://github.com/aniruddh-jammoria/obsidian-agentic-todolist)** [![GitHub stars](https://img.shields.io/github/stars/aniruddh-jammoria/obsidian-agentic-todolist?style=flat&color=gold)](https://github.com/aniruddh-jammoria/obsidian-agentic-todolist) - An Obsidian plugin that turns a Markdown todo file into a two-way-synced, Google-Tasks-style kanban board designed to pair with an AI agent.
- **[fable-global-skills](https://github.com/aniruddh-jammoria/fable-global-skills)** - A library of Claude Code Agent Skills covering a project's full lifecycle from spec to announcement, each one distilling how a frontier model handles the task into explicit instructions smaller models can follow reliably.
- **[eval-dual-GPU](https://github.com/aniruddh-jammoria/eval-dual-GPU)** - A benchmarking dataset comparing Ollama vs. llama.cpp across 8 models and 5 GPU split configurations, measuring decode/prefill speed, latency, and power draw to find the inference parameters (backend, tensor vs. layer split) that actually maximize tok/s on multi-GPU setups.
- **[research-newsletter-pipeline](https://github.com/aniruddh-jammoria/research-newsletter-pipeline)** - A configurable pipeline that generates a weekly research/news newsletter as a PDF and delivers it via Telegram, with LLM-based filtering and dedup.

Other projects: 
- **[gpt-from-scratch](https://github.com/aniruddh-jammoria/gpt-from-scratch)** - A from-scratch, notebook-based build of GPT-style language models following Karpathy's "Let's build GPT," from a bigram baseline to a full transformer.
- **[search-api-eval](https://github.com/aniruddh-jammoria/search-api-eval)** - A benchmarking framework for comparing web search API providers, scoring results with two independent LLM judges and outputting a self-contained HTML report.
